# Support

Need help with GitDocAI? Here's how to get support.

## Documentation

Start here for guides, tutorials, and API references:

- [Official Documentation](https://docs.gitdoc.ai)
- [Getting Started Guide](https://docs.gitdoc.ai/v1.0.0/documentation/getting_started/introduction)

## Community Support

### GitHub Discussions

The best place to get help from the community:

- **[Q&A Discussions](../../discussions/categories/q-a)** - Ask questions and get answers
- **[General Discussions](../../discussions/categories/general)** - General topics and discussions

**Before posting:**
1. Search existing discussions to see if your question has been answered
2. Check the documentation
3. Provide clear details about your issue

### Tips for Getting Good Help

When asking for help, include:

- Clear description of what you're trying to do
- What you've already tried
- Error messages (full text)
- Your environment (OS, GitDocAI version, etc.)
- Minimal example if possible

## Report Bugs

Found a bug? Please report it:

1. Check if it's already reported in [Issues](../../issues)
2. If not, [create a bug report](../../issues/new?template=bug_report.yml)

## Request Features

Have an idea for a feature?

1. Search [Ideas discussions](../../discussions/categories/ideas) to see if it exists
2. If not, [start a new discussion](../../discussions/new?category=ideas)

## Professional Support

Need dedicated support or consulting?

- Email: support@gitdoc.ai

## Security Issues

Found a security vulnerability?

**DO NOT** open a public issue. See our [Security Policy](SECURITY.md) for how to report it responsibly.

## Stay Updated

- [Announcements](../../discussions/categories/announcements) - New releases and updates
- [Roadmap](../../projects) - See what's coming

## Response Times

- **Community Support** (Discussions): Best effort from community members
- **Bug Reports**: We review all reports and prioritize based on severity
- **Security Issues**: We aim to respond within 48 hours
